**Step 3 -- Create `lib/print_uint64.asm` (per D-02, D-05):**
Create directory `lib/` at project root. Create `lib/print_uint64.asm` implementing a routine that:
- Takes a 64-bit unsigned integer in `rdi` (System V convention)
- Converts it to decimal ASCII string
- Prints it to stdout via `sys_write` (syscall 1) followed by a newline (`0x0a`)
- Uses `_start`-style syscalls (no libc) per D-01
- Exports symbol: `global print_uint64`

Implementation approach (div loop):
1. Allocate a 21-byte buffer on the stack (20 digits max for UINT64_MAX + newline)
2. Start from the end of the buffer, place newline byte `0x0a`
3. Repeatedly divide `rdi` by 10, convert remainder to ASCII (`add '0'`), store in buffer moving backward
4. Handle special case: if input is 0, output `"0\n"`
5. After conversion, call `sys_write` (rax=1, rdi=1, rsi=buffer_start, rdx=length)
6. Return via `ret`
7. Clobber registers: rax, rcx, rdx, rsi, rdi, r8 (documented in header comment)

The header comment MUST include the interface contract:
```nasm
; print_uint64 -- print a 64-bit unsigned integer to stdout
;
; Input:  rdi = 64-bit unsigned integer to print
; Output: integer printed to stdout as decimal, followed by newline
; Clobbers: rax, rcx, rdx, rsi, rdi, r8, r11 (syscall clobbers rcx, r11)
```

Verify the helper assembles: `nasm -f elf64 -g lib/print_uint64.asm -o lib/print_uint64.o` -- must produce `lib/print_uint64.o` with no errors.

The exercise demonstrates:

- `mov ax, imm16` -- loading a 16-bit immediate, upper bits UNCHANGED
- `mov al, imm8` -- loading an 8-bit immediate, upper bits UNCHANGED

The program prints descriptive string labels to stdout using sys_write for each demonstration. For example:
```

Loading eax (clears upper 32 bits of rax)
Loading ax (preserves upper bits)
Loading al (preserves upper bits)
Register-to-register: mov rbx, rax
--- done ---
```

Each section has heavy inline comments explaining what happens at the hardware level. Include a comment block about the zero-extension rule (per Pitfall 6): "Writing to eax ALWAYS clears the upper 32 bits of rax. Writing to ax or al does NOT."
