**File 1: `01-registers/asm-02-arithmetic/main.asm` (per ASM-02, D-01, D-02, D-06, D-07)**

A complete NASM x86-64 program demonstrating all four arithmetic operations. The program:

1. Uses `extern print_uint64` to access the shared helper
2. Uses `global _start` and `_start:` entry point
3. All output via sys_write (strings) and print_uint64 (numbers) -- no libc (per D-01)

The exercise must demonstrate these operations with concrete, verifiable values:
- `add`: e.g., `mov rax, 30` / `add rax, 12` / print result (42)
- `sub`: e.g., `mov rax, 100` / `sub rax, 58` / print result (42)
- `imul`: e.g., `mov rax, 6` / `mov rbx, 7` / `imul rax, rbx` / print result (42)
- `idiv`: e.g., `mov rax, 17` / `mov rcx, 5` / `cqo` / `idiv rcx` / print quotient (3) and remainder (2)

For each operation, print a descriptive label string via sys_write, then the numeric result via print_uint64. Example output format:
```
--- arithmetic operations ---
add: 30 + 12 =
42
sub: 100 - 58 =
42
imul: 6 * 7 =
42
idiv: 17 / 5 = quotient:
3
idiv: 17 % 5 = remainder:
2
--- done ---
```

**CRITICAL: idiv section** must include `cqo` before `idiv` per Pitfall 3. Include an inline comment: "; CRITICAL: cqo sign-extends rax into rdx:rax. Without this, idiv uses garbage in rdx and produces wrong results or SIGFPE."

**CRITICAL: register preservation around print_uint64 calls.** print_uint64 clobbers rax, rcx, rdx, rsi, rdi, r8, r11. If a value is needed after the call (e.g., rdx=remainder after idiv), it must be saved to a callee-saved register (rbx, r12-r15) or pushed to stack before the call.

Per-exercise Makefile WITH lib/ dependency (per Pattern 1 from RESEARCH.md):
```makefile
ASM      = nasm
LD       = ld
ASMFLAGS = -f elf64 -g

SRC      = main.asm
OBJ      = main.o
LIB_OBJ  = ../../lib/print_uint64.o
BIN      = main

.PHONY: build run check clean

build: $(BIN)

$(BIN): $(OBJ) $(LIB_OBJ)
	$(LD) -o $@ $^

$(OBJ): $(SRC)
	$(ASM) $(ASMFLAGS) -o $@ $<

$(LIB_OBJ): ../../lib/print_uint64.asm
	$(ASM) $(ASMFLAGS) -o $@ $<

run: build
	./$(BIN)

check: build
	./$(BIN) > actual.txt
	diff -u expected_output actual.txt && echo "PASS" || (echo "FAIL"; rm -f actual.txt; exit 1)
	@rm -f actual.txt

clean:
	rm -f $(OBJ) $(BIN) actual.txt
```

The exercise must demonstrate at least these conditional jump types:
- `je` (jump if equal / ZF=1)
- `jne` (jump if not equal / ZF=0)
- `jl` (jump if less / SF != OF, signed)
- `jg` (jump if greater / ZF=0 and SF=OF, signed)

Structure: The program compares pairs of values and branches to print which branch was taken. For example:

```
--- conditional jumps ---
Comparing 10 and 10: equal
Comparing 10 and 20: not equal
Comparing 5 and 10: less than
Comparing 20 and 10: greater than
--- done ---
```

Each comparison is a `cmp reg, reg` followed by a `jcc` to a label that does sys_write of the appropriate result string. Include inline comments explaining:
- `cmp` computes `rax - rbx` internally, sets FLAGS, discards the result
- Each jcc tests specific FLAGS bits (document which: ZF for je/jne, SF and OF for jl/jg)

Use a fall-through + jump pattern to show both branches for at least one comparison (e.g., for the "equal" test, show what happens when values are NOT equal -- the je does not jump, execution falls through to the "not equal" path).

A complete NASM x86-64 program demonstrating a counted loop using a register counter. The program:

1. Uses `extern print_uint64` for numeric output
2. Uses `global _start`, sys_write + sys_exit only (per D-01)
3. Demonstrates `dec` + `jnz` loop pattern (NOT the legacy `loop` instruction -- per RESEARCH.md recommendation)

The exercise:
- Initializes a counter in a callee-saved register (e.g., `r12` or `rbx`) to avoid clobbering by print_uint64
- Loops N times (e.g., 5), printing the current iteration number each time via print_uint64
- Uses `dec reg` + `jnz` as the loop mechanism (not the `loop` instruction)
- Includes a header and footer string via sys_write

**CRITICAL: Register preservation.** Since print_uint64 clobbers rax, rcx, rdx, rsi, rdi, r8, r11, the loop counter MUST be in a callee-saved register (rbx, r12, r13, r14, r15) or pushed/popped around each call. Use a callee-saved register (e.g., `r12` for counter, `r13` for current iteration value) for clarity.

Example output (counting up from 1 to 5):
```
--- loop: counting 1 to 5 ---
1
2
3
4
5
--- done ---
```

Include inline comments explaining:
- Why the counter is in r12 (callee-saved, not clobbered by print_uint64)
- How `dec` sets ZF when the counter reaches zero
- Why we use `dec + jnz` instead of the `loop` instruction
- The note: "We pass the number in rdi -- you'll learn exactly why in Phase 2"
